closure(闭包)
what is a closure
闭包是一个可以访问外部作用域的内部函数。
闭包是函数和声明该函数的词法环境的组合。这个环境包含了这个闭包创建时所能访问的所有局部变量

使用闭包：在函数中创建内部函数并传出
作用域链： 闭包函数的作用域中保存着[当前Context.AO, 父类Context.AO, 全局Context.VO], Context.AO 指向 活跃对象;
注意： 父类函数调用结束，其执行上下文对象销毁，但是父类作用域对象仍是存在的，因为其让被闭包函数的作用域链所引用, 垃圾回收算法中利用 标记清除 算法会保存该作用域对象。
注： 执行上下文对象(context) 和 作用域对象（scope) 并非同一对象

  • 给对象提供隐私数据

this
this一般指向调用该函数的对象，一般分为几种情况：

  • 默认绑定(非严格模式指向全局对象, 严格模式下指向 undefined)
  • 作为对象方法的调用, 赋值操作(全局赋值和参数传递)会破坏该隐式绑定
  • 构造函数的实例对象调用
  • call/apply 调用(显示绑定), bind 永久绑定, 传入null 或 undefined 会被忽略，应用默认规则
  • 箭头函数的this由 外层作用域 来决定，本身不绑定

类和原型继承的区别
What’s the Difference Between Class & Prototypal Inheritance

继承基本上是一种代码重用机制：一种让不同类型的对象共享代码的方法
js是基于原型继承的。JavaScript类继承使用原型链将子Constructor.prototype连接到父Constructor.prototype以进行委派。

ES6的‘class’关键字 是构造函数的语法糖

类继承在OO设计中导致的问题：

  • 紧耦合性问题
  • 脆弱的基类问题
  • 不灵活的层次结构问题
  • 必要的重复问题

三种原型继承：

  • Concatenative inheritance(连接继承)：通过复制源对象属性将要素直接从一个对象继承到另一个对象的过程; eg: Object.assign(), …
  • Prototype delegation(原型委托)：如果在对象上找不到属性，则将查找委托到proto指向的委托原型上，依此类推，直到Object.prototype为止。eg: Object.create()
  • Functional inheritance(功能继承)：通过函数生成对象
    委托原型不是类继承的绝佳替代 - 对象组合才是。

js继承
四种继承方式：

  • 原型链继承： child.prototype = new parent(); 缺点：共享了父类的所有的属性和方法，有些属性需子类特有，或者不需要共享
  • 构造函数继承： 利用 call或apply继承父类属性，子类各有一份属性，修改不会影响； 缺点： 无法继承原型属性/方法
  • 组合继承：上两种继承的结合，注意需修改子类构造函数的指向(constructor); 缺点：父类构造函数执行了两次，造成内存浪费
  • 寄生组合继承： 修复组合继承的小bug,通过生成一个 原型副本

promise
what is a Promise
Promise 是一个可能在未来某个时间产生单一值的对象.
其中单一值是resolved值或者rejected值，promise有三种状态： fulfilled, rejected, pending,状态转变之后将不可改变，then可以通过callback处理单一值。
promise的 .then 方法总是返回一个 new promise,以便 链式调用,调用中 rejected被执行之后，不会继续向下传递, 需要 .catch 捕抓 handleSuccess的错误

setState机制
深入理解setState
setState 通过一个队列机制来实现 state 更新，当执行 setState() 时，会将需要更新的 state 浅合并后放入 状态队列，而不会立即更新 state，队列机制可以高效的批量更新state.
可通过 (prevState, props) => newState 的函数作为参数传递。
batch update(批量更新)依据 isBatchingUpdates 判断是否批量更新， 在 batchedUpdates 方法中，执行 transaction.perform调用
