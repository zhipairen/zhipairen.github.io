路由

路由： 检测url变化，解析url来匹配路由规则。 注： URL到函数的映射

https://www.zhihu.com/question/53064386
前后端路由区别
后端路由：web服务接收到请求，解析url路径，然后请求对应的资源。注：web服务不会解析hash
前端路由： 由浏览器控制，js解析控制，响应不同路径变化，注：无刷新，spa应用
目前实现前端路由的两种方式：

  • 通过改变 hash 值，监听 onhashchange事件，兼容低版本浏览器
  • 通过 history对象，监听 popState事件，有pushState和replaceState方法，为会话历史栈添加路由信息。
    注：history 的两个api可以改变url地址且不会发送请求，但当主动刷新页面时，还是会向服务器发送请求，所以需要服务器把所有路由重定向到根页面

利用 script 元素的开放策略，可获得其他来源的js。
在该元素上的src属性，设置回调函数名称在服务器请求的查询参数上，服务器将数据填充到回调函数中

服务器端返回这样的js给客户端

service worker 服务工作线程

google service worker
它支持离线体验。 相关事项：

  • 是一种js工作线程，无法直接访问 DOM；它通过响应 postMessage 接口发送的信息与页面通信
  • 是一种可编程的网络代理
  • 它在不用时会被中止，下次有需要时重启，若要保存重启后加以重用的信息，它可以访问 IndexedDB API
  • 已广泛使用 promise
