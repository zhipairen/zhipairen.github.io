get和post区别

https://www.oschina.net/news/77354/http-get-post-different
底层都是 TCP 实现，在 HTTP 协议约束下，发出请求处理

  • GET产生一个TCP数据包；POST产生两个TCP数据包(先发header，后发 data)
  • GET请求限制url长度在 2k 个字节，POST无参数大小限制
  • GET请求只能进行 url 编码， POST支持多种编码方式

es6重点

发表于 2018-11-21 | 分类于 es6

Module

es6模块和commonjs模块的差异
重大差异：

  • CommonJS模块输出的是值的拷贝(内存中缓存),ES6模块输出的是值的引用(只读)
  • CommonJS模块是运行时加载(加载完模块才会生成对象),ES6模块是编译时输出接口(js引擎对脚本静态分析时，遇到import生成只读引用)。

commonjs模块

  • CommonJS 模块重复引入的模块并不会重复执行，再次获取模块直接获得暴露的 module.exports 对象
  • 如果你要处处获取到模块内的最新值的话，也可以你每次更新数据的时候每次都要去更新 module.exports 上的值
  • 如果你暴露的 module.exports 的属性是个对象，那就不存在这个问题了
    es6静态编译
  • import 命令会被 JavaScript 引擎静态分析，优先于模块内的其他内容执行
  • export 命令会有变量声明提前的效果

后端知识整理集

小知识点

  • CPU多级缓存：L1 Cache(12k)、L2 Cache(256k)、L3 Cache(12M),级别越小越接近CPU，所以速度也更快，同时也代表着容量越小。
  • 缓存是由缓存行组成的，一般一行64字节，是cpu操作最小单位

JWT

阮一峰 jwt
JWT token组成：Header(头部).Payload(负载).Signature(签名),由三部分的数据经Base64Url转成字符串组合成

JWT 本身包含了认证信息，存储在客户端token中, 一旦泄露token，任何人都可获得该令牌的所有权限。设置主动过期，并使用 HTTPS协议传输

OAuth 开放授权
简单步骤：

  1. 第三方应用请求导向认证服务，获得授权许可（重定向之后携带code)
  2. 根据code向认证服务获取授权token(access_token)
  3. 根据授权token，去获取用户已授权可获取的资源
