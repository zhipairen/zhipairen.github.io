es6重点

发表于 2018-11-21 | 分类于 es6

Module

es6模块和commonjs模块的差异
重大差异：

  • CommonJS模块输出的是值的拷贝(内存中缓存),ES6模块输出的是值的引用(只读)
  • CommonJS模块是运行时加载(加载完模块才会生成对象),ES6模块是编译时输出接口(js引擎对脚本静态分析时，遇到import生成只读引用)。

commonjs模块

  • CommonJS 模块重复引入的模块并不会重复执行，再次获取模块直接获得暴露的 module.exports 对象
  • 如果你要处处获取到模块内的最新值的话，也可以你每次更新数据的时候每次都要去更新 module.exports 上的值
  • 如果你暴露的 module.exports 的属性是个对象，那就不存在这个问题了
    es6静态编译
  • import 命令会被 JavaScript 引擎静态分析，优先于模块内的其他内容执行
  • export 命令会有变量声明提前的效果

计算机基础知识

发表于 2018-10-12 | 分类于 计算机基础

DNS服务器、IP地址、MAC地址

掘金
MAC地址

  • 计算机建立连接是网卡之间建立连接
  • 网卡地址由供应商提供，全球唯一
  • 网卡由48位的二进制组成： 24bite(供应商标识)+24bite(供应商提供的编号)

IP地址

  • 由INternet分配机构统一分配，Ip地址标识网络中的一个节点
  • IP地址由32位二进制组成，网络标识区分A,B,C类，IP=网络标识+主机号
  • IP为255.255.255.255称为限制广播地址，0.0.0.0表示启动时Ip,127开头的代表本机
  • A类子网掩码：255.0.0.0 B类：255.255.0.0 C类：255.255.255.0；IP地址和子网掩码位与得到网络标识；能否相互通信，ping命令的网络标识应一样
    浏览器只对网络请求有同源限制，同源就是协议、域名和端口号一致

TCP

TCP传输控制协议：会将数据分段打包传输，并对每个数据包编号控制顺序，传输中出现丢包时，会有重发等机制处理保证数据完整性

三次握手：

第一次握手主机A通过一个标识为SYN标识位的数据段发送给主机B请求连接，通过该数据段告诉主机B希望建立连接，需要B应答，并告诉主机B传输的起始序列号
第二次握手是主机B用一个确认应答ACK和同步序列号SYNC标志位的数据段来响应主机A，一是发送ACK告诉主机A收到了数据段，二是通知主机A从哪个序列号做标记。
第三次握手是主机A确认收到了主机B的数据段并可以开始传输实际数据。

第一次握手主要是确定服务端确认客户端能够发送信号；第二次握手主要是客户端确认服务端能够接收和发送信号；第三次握手主要是服务端确认客户端能够接收信号
四次挥手：

主机A发送FIN控制位发出断开连接的请求
主机B进行响应，确认收到断开连接请求
主机B提出反方向的关闭要求
主机A确认收到的主机B的关闭连接请求

第一次挥手是服务端确认客户端需要断开连接；第二次挥手是客户端确认服务器接收断开请；第三次挥手是客户端确认服务器数据发完，断开连接；第四次挥手是服务端确认客户端断开连接，断开连接。所以如果服务端的数据全部发送完，是没有第三次挥手，直接进入第四次挥手。

深入理解React高阶组件

Higher Order Components

高阶组件HOC应用

  1. 当装饰器使用：@higherOrderComponent
  2. 传递参数,定义HOC工厂函数，返回一个HOC： @higherOrderComponent(args)
  3. 反向继承HOC: function(WrappedComponent){ return class extends WrappedComponent { ….. }}
    注意：
  • 不要在render函数中使用高阶组件，若使用的话，每次都将创建一个新的组件实例，则使用差异算法时，每次都将子对象树完全移除，*浪费性能，可能引起原组件状态和子组件丢失。
  • 当使用高阶组件包装组件，原始组件被容器组件包裹，也就意味着新组件会丢失原始组件的所有静态方法，必须将静态方法做拷贝
  • 不能传递REfs属性引用,如果你向一个由高阶组件创建的组件的元素添加ref应用，那么ref指向的是最外层容器组件实例的.

render prop
render prop 是一个组件用来了解要渲染什么内容的函数 prop;方便了共享代码，复用行为（eg: 共享鼠标坐标)
注： 会抵消了继承自React.PureComponent的效果

code-splitting

Refs
Refs 提供了一种方式，用于访问在 render 方法中创建的 DOM 节点或 React 元素。
ref 中的 current 属性对节点的引用进行访问。(注：函数式组件无实例，所以不能使用ref属性).
两种实现方式： 回调函数(定义成类的绑定函数的方式)、React.createRef()；ref 回调函数会在 componentDidMout 和 componentDidUpdate 生命周期函数前被调用。

不可变数据结构：Immutable
避免使用值(较深层级的对象)可能突变的情况，通过结构共享提供不可突变的，持久集合。

  • 不可突变:一旦创建，集合就不能在另一个时间点改变。
  • 持久性:可以使用原始集合和一个突变来创建新的集合。原始集合在新集合创建后仍然可用。
  • 结构共享:新集合尽可能多的使用原始集合的结构来创建，以便将复制操作降至最少从而提升性能。
    不可突变数据使得变化跟踪很方便，每个变化都会导致产生一个新的对象。
    注： createReactClass与 mixins混入使用，es6中不可用。
    组件可以接受任意元素，包括基本数据类型、React 元素或函数

揭秘 SSR

客户端渲染： 页面在js(Bunlde),css等资源文件加载完之后渲染页面，路由为客户端路由(SPA)
服务端渲染： 页面由服务端直接返回浏览器，路由为服务端路由，URL变更刷新页面
同构(Universal)：js代码可同时运行在浏览器及node中，由服务端渲染提升首屏加载速度，首屏之后路由由客户端控制，整个应用依然是SPA
