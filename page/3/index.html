解释Node

what exactly is Node.js

Node.js是基于Chrome的V8 JavaScript引擎构建的JavaScript运行时,使用事件驱动的非阻塞I/O模型，使其轻量且高效。

understanding of Buffer, Streams, Binary data
更简单的理解： Buffer(缓冲) 、 Streams(流)、 binary data(二进制数据)

字符，图像，视频都有关于如何转换或编码将其存储在二进制文件中的规定规则，在计算机中所有的数据类型都存储在二进制文件中，称为二进制数据

  • 字符集： Unicode, ASCII（字符映射的数字，eg: L => 76);
  • 字符编码：UTF-8 (一个字节由几位bits,eg: 1Byte = 8 bit)

流：Node.js中的流只是意味着一段数据随着时间的推移从一个点移动到另一个点。整个概念是，您需要处理大量数据，但在开始处理之前无需等待所有数据可用。

通常，数据的移动通常是为了处理或读取数据，并基于它做出决策。但是，随着时间的推移，流程可能需要的最小和最大数据量。
因此，如果数据到达的速率比进程消耗数据的速率快，则多余的数据需要在某处等待轮流进行处理;
另一方面，如果进程消耗的数据比它到达的速度快，那么早先到达的少数数据需要等待一定量的数据到达才能被发送出去进行处理。
那个“等候区”就是缓冲区！它是计算机中的一个小型物理位置，通常位于RAM中，数据在时间上被收集，等待，并最终在流式传输期间发送出去进行处理。

我们可以将整个流和缓冲过程视为公交车站。在一些公交车站，在一定数量的乘客到达或直到特定的出发时间之前，不允许公共汽车离开。
此外，乘客可以以不同的速度到达不同的时间。乘客和公交车站都无法控制乘客到达车站。

无论如何，总会有一个等待的地方。那是Buffer to Node.js！ Node.js无法控制数据到达的速度或时间，流的速度。它只能决定何时发送数据。如果还没有时间，
Node.js会将它们放入缓冲区(buffer) - “等待区” - RAM中的一个小位置，直到将它们发送出去进行处理

系列二： V8引擎内部&&如何优化代码的5个技巧

原文出处

概念

js引擎是执行js代码的程序或解释器。
其中有谷歌开发的 V8 引擎，开源项目，由C++ 编写而成，被应用在chrome 和 node中。
为了运行速度，V8转换 js 代码为更高效的 机器代码，它通过 Just-In-Time(JIT)即时编译器将js编译为机器码，不产生字节码或任何中间代码。
注：最新V8引擎优化，加入了转字节码处理

v8在版本5.9以前，主要编译器为： full-codegen(简单快速编译产生机器码), Crankshaft(优化编码)

v8内部

经常使用的多个线程有以下：

  1. 主线程：获取代码，编译代码然后解析代码
  2. 有一个单独的线程用于编译，因此主线程可以继续执行，前者可优化代码
  3. 一个Profiler线程，它监控在运行时花费大量时间的方法，以便Crankshaft优化它们
  4. 一些线程处理 垃圾收集器扫描（GC)

首次执行js代码时，V8利用 full-codegen 编译器直接翻译解析js代码为机器码不需要转换，执行速度快；v8没有使用中间字节码，不需要解释器；
当代码运行一段时间之后， profiler 线程收集了足够的数据告诉那些方法应该优化；
然后，Crankshaft 开始在另一个线程优化，它将js的抽象语法树（AST)转换为名为Hydrogen的 高级的静态 SSA表示，然后尝试优化它。

隐藏类
js是以原型链为基础的语言，没有 class;
同时 js 是一门动态编程语言，这意味着在实例化后，可以方便地添加或删除属性。

大部分的js解释器利用 字典结构在内存中存储对象属性值；这种结构在检搜属性值比非 动态语言像java或c需要更多的计算；
静态语言的属性值在内存中容易得到 偏移量，从而快速找到，但js不行。
由于字典结构查找效率低，所以V8用 隐藏类代替，隐藏类存放对象属性的内存位置信息，类似于固定了对象布局(classes)

Object的内存结构与属性访问
在对象中创建新属性，V8会基于初始隐藏类C0,创建隐藏类C1替换C0,在C1中存放属性的内存位置信息；注： 对象会重复使用隐藏类；
对于对象上的方法，不在隐藏类中存放函数引用，而是映射为 COnstant Function 描述符，对象方法建议写在原型上；
对于复杂属性的对象，v8会采用字典模式来存储对象，也就是哈希表来存放键值信息

内联缓存
如果v8能够很好的确定使用过的对象类型，那么它可以跳过找出对象属性的步骤，利用之前存储的对象里的隐藏类信息。
在特定对象上调用方法时，v8必须执行该对象隐藏类的查找，以确定访问特定属性的偏移量;
在将同一方法成功调用两次到同一隐藏类后，v8省略隐藏类查找，将属性的偏移量添加到对象指针本身上，在该方法的未来调用中，v8假定隐藏类未改变，
并使用先前查找到的偏移量直接跳到特定属性的内存地址，大大提高执行速度。

GC 垃圾回收
对于垃圾回收，V8采用传统的标记和扫描方式清理旧一代，标记阶段会停止js执行，所以V8使用增量标记：
不变量整个堆，尝试标记每个可能对象，只遍历堆的一部分，然后恢复正常执行；下一个GC从上一个堆行走停止的位置继续，扫描阶段由单独线程处理

Ignition and TurboFan
新版本之后，新的执行管道建立在Ignition,V8解释器,TurboFan;
v8加入了字节码的中间代码转换，减小了内存使用，利用时间换空间，为进一步优化铺平了道路

如何写优雅的js

  1. 对象属性的顺序： 相同的顺序方便隐藏类的顺序，加快查找
  2. 动态属性：在实例化后的对象中添加属性会强制影响之前隐藏类优化的任何方法(offset不一致了)，所以在其构造函数中分配所有对象属性
  3. 方法： 重复执行相同的方法代码会比只执行一次的运行的快，由于内联缓存
  4. 数组： 避免不是增量数的稀疏数组；避免预先分配大型数组，不要随便删除数组元素
  5. 标记值： V8用32位表示对象和数字，它使用一位（flag=1)知道为对象，flag=0称为SMI的整数

系列三： 内存管理+如何处理4种常见的内存泄漏

概念

像C这样的语言，提供了原始的内存管理函数 malloc() 和 free()，开发者可通过这些方法从操作系统直接分配和清空内存；
同时， js在创建事务（对象，字符等）时会分配内存，当不再使用时，“自动”清理它们，这个过程叫“垃圾回收”

什么是内存

生命周期
分配内存 ==> 使用内存 ==> 释放内存
概念
可以将整个计算机内存视为可以读写的一个巨大的位数组；
有很多东西存储在内存中：

  1. 所有程序使用的所有变量和其他数据
  2. 程序的代码，包括操作系统

编译器和操作系统一起管理内存：
当编译代码时，编译器可检查原始数据类型并提前计算它们需要多少内存，然后将所需数量传递给调用堆栈空间(stack space)的程序。
分配这些变量的空间称为堆栈空间，在调用函数式，它们的内存会添加到现有内存上，终止时，以先进先出的顺序移除。

当函数调用其他函数时，每个函数都会获得自己的堆栈，将所有的局部变量保存在那里，还有个程序计算器记录执行位置，当函数完成时，其内存块被释放。

动态分配

当我们在编译时不知道变量需要多少内存时，事情就不那么容易了。比如数组，需要根据传进来的具体值才能确定需要分配多少内存，因此它无法为堆栈分配空间，
相反，程序需要在运行时明确询问操作系统是否有适当的空间量，此内存是从堆空间(heap space)分配的.

js四种情况容易导致内存泄漏

  1. 滥用全局变量： 在垃圾回收时，需主动设置变量为 null,来释放内存
  2. 定时器或回调忘记处理： 现代浏览器已做该项优化或主动删除
  3. 闭包：闭包指的是内部函数可以访问外部函数的变量；（eg: 循环引用、有关于全局变量的引用）
  4. DOM的外部引用： 当把DOM变量存储在对象中，会有两处引用，一处是Dom树，另一处是字典对象，只移除dom树的元素，容易造成内存泄漏

了解v8的垃圾回收算法

解读 V8 GC(1)
解读 V8 GC(2)
垃圾回收器将对象分为新生代和老生代，新生代采用Scavenge回收策略，经历两次GC没有被回收的对象会晋升到老生代，
老生代采用 Mark-Sweep-Compact回收策略。
对象都按照 4 字节（32 位机器）或者 8 字节（64 位机器）对齐，判断最后一位是 0 还是 1，来直接分辨两种类型，1 表示指针，0 表示整数；
准确式GC,尽早区分出SMI结构。
函数创建的对象有很高存活率，将会直接在 old space分配，省略了这些对象在 New Space 第一次 GC 的时候大量复制到另一个 semispace，第二次 GC 又大量复制到 Old Space 的开销。即使猜错了，反正下一次老生代 GC 的时候这些对象也会被回收走。

编码（encoding)

概念
编码本质上是 信息形式的转化,可通过解码还原原始信息；
目的
将信息转化为统一的格式，方便在不同系统之间传输。
常见列子
ASCII、Unicode、 URL ENcoding、Base64

摘要(digest)

概念
摘要是通过 散列函数对信息运算生成固定大小的’指纹’,不可被还原。
目的
相同信息生成的摘要总是相同的，所以可校验信息的完整性，防止信息被篡改。
常见例子
MD5(单向散列算法), SHA(安全散列算法), MAC(信息认证代码)
散列算法：不可逆，防冲突

加密(encryption)

概念
加密是 将明文信息转为难以读取的密文内容，使其不可读, 可通过密钥还原
目的
加密是为了保证信息安全，无正确密钥的无法解密出信息内容

加密算法

  • 分为两类：对称加密和非对称加密(速度更慢)。
  • 常见加密算法
    对称加密
    加密和解密均采用同一密钥，通信双方都需要该钥匙。
    常见算法: DES、3DES、AES(高级加密标准)

非对称加密
加密采用公钥，解密采用私钥，是一对不同的钥匙。
常见算法: RSA(因子分解)、DSA(数字签名)、ECC(椭圆曲线密码更优)

注意：通常采用非对称算法管理对称算法的密钥，用对称算法加密数据，这样既实现对称加密速度快的优点，又实现安全管理对称密钥的优点
公开密钥密码体系
证书：将公钥和公钥主人联系在一起，再请权威机构（CA)确认，加上该权威机构的签名(分配的公钥和主人信息经机构私钥加密)，形成证书。
CA: 签发证书(可信任的签字)，提供一个可信任的公钥，还是证书和密钥的管理

浏览器需要验证网站是否真实可靠，服务器发送数字证书给浏览器；
证书内容大致有：

  • 签发者
  • 证书用途
  • 自己的公钥
  • 自己的加密算法，Hash算法
  • 证书到期时间等
    将证书内容做HASH处理得到固定长度的字符，再用CA的私钥加密，得到数字签名；
    浏览器根据CA公钥值，解密数字签名，得到原始的HASHs,然后自己也按照证书的HASH算法，计算一个HASHc,如果HASHc == HASHs，则认证通过；
    通信双方利用DH算法，协商一个 master-key,以master-key推导出session-key，用于双方SSL数据流的加密/解密,采用对称加密(AES)；
    以master-key推导出 hash-key，用于数据完整性检查的密钥，HASH算法：MD5、SHA,生成MAC,保证数据不被篡改；
    附在HTTP报文的后面，用 session-key加密所有数据（HTTP+MAC）；

计算机网络的可靠支付：SSL(现以为TLS)负责业务层面的可靠， TCP负责传输层面的可靠

概念

把ES6+代码转译（转码+编译) 到es5(语法层次);它将运行代码分为3个阶段：解析（Parse)，转化(Transform)，生成(Generate)；

  • 解析：词法分析将代码转化为令牌流，语法分析将令牌流转化为AST表示
  • 转化: 遍历令牌流，根据配置对节点增，更，移等操作
  • 生成： 将经过转化的抽象语法树（AST)重新生成代码，并创建SourceMap

注意: babel只转换新语法，需要polyfill,babel是一个js编译器;

  • plugins: es6+新语法(箭头函数,class)转译
  • polyfill(垫片)： es6+新增的api(reset,promise等)无法转译，需要引入polyfill兼容api

    基础建设

  • 安装 babel cli 和env preset

  • 创建 .babelrc文件（或package.json增加babel属性）

  • babel-runtime: 内部集成了 1. core-js(内置了Promise,Symbols等类 和静态方法Array.from等) 2. regenerator(generator/yield和async/await的支持) 3. helpers

前后端路由区别
后端路由：web服务接收到请求，解析url路径，然后请求对应的资源。注：web服务不会解析hash
前端路由： 由浏览器控制，js解析控制，响应不同路径变化，注：无刷新，spa应用
目前实现前端路由的两种方式：

  • 通过改变 hash 值，监听 onhashchange事件，兼容低版本浏览器
  • 通过 history对象，监听 popState事件，有pushState和replaceState方法，为会话历史栈添加路由信息。
    注：history 的两个api可以改变url地址且不会发送请求，但当主动刷新页面时，还是会向服务器发送请求，所以需要服务器把所有路由重定向到根页面

google service worker
它支持离线体验。 相关事项：

  • 是一种js工作线程，无法直接访问 DOM；它通过响应 postMessage 接口发送的信息与页面通信
  • 是一种可编程的网络代理
  • 它在不用时会被中止，下次有需要时重启，若要保存重启后加以重用的信息，它可以访问 IndexedDB API
  • 已广泛使用 promise
