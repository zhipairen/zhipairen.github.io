深入理解React高阶组件

Higher Order Components

高阶组件HOC应用

  1. 当装饰器使用：@higherOrderComponent
  2. 传递参数,定义HOC工厂函数，返回一个HOC： @higherOrderComponent(args)
  3. 反向继承HOC: function(WrappedComponent){ return class extends WrappedComponent { ….. }}
    注意：
  • 不要在render函数中使用高阶组件，若使用的话，每次都将创建一个新的组件实例，则使用差异算法时，每次都将子对象树完全移除，*浪费性能，可能引起原组件状态和子组件丢失。
  • 当使用高阶组件包装组件，原始组件被容器组件包裹，也就意味着新组件会丢失原始组件的所有静态方法，必须将静态方法做拷贝
  • 不能传递REfs属性引用,如果你向一个由高阶组件创建的组件的元素添加ref应用，那么ref指向的是最外层容器组件实例的.

Refs
Refs 提供了一种方式，用于访问在 render 方法中创建的 DOM 节点或 React 元素。
ref 中的 current 属性对节点的引用进行访问。(注：函数式组件无实例，所以不能使用ref属性).
两种实现方式： 回调函数(定义成类的绑定函数的方式)、React.createRef()；ref 回调函数会在 componentDidMout 和 componentDidUpdate 生命周期函数前被调用。

不可变数据结构：Immutable
避免使用值(较深层级的对象)可能突变的情况，通过结构共享提供不可突变的，持久集合。

  • 不可突变:一旦创建，集合就不能在另一个时间点改变。
  • 持久性:可以使用原始集合和一个突变来创建新的集合。原始集合在新集合创建后仍然可用。
  • 结构共享:新集合尽可能多的使用原始集合的结构来创建，以便将复制操作降至最少从而提升性能。
    不可突变数据使得变化跟踪很方便，每个变化都会导致产生一个新的对象。
    注： createReactClass与 mixins混入使用，es6中不可用。
    组件可以接受任意元素，包括基本数据类型、React 元素或函数

react SSR
揭秘 SSR

客户端渲染： 页面在js(Bunlde),css等资源文件加载完之后渲染页面，路由为客户端路由(SPA)
服务端渲染： 页面由服务端直接返回浏览器，路由为服务端路由，URL变更刷新页面
同构(Universal)：js代码可同时运行在浏览器及node中，由服务端渲染提升首屏加载速度，首屏之后路由由客户端控制，整个应用依然是SPA
