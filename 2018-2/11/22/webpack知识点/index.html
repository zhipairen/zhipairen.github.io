优化

模块标识符： 当增加新的模块时, vendor的hash会发生变化（默认module.id),优化可使用HashedModuleIdsPlugin，保证每次构建，vendor hash保持一致

抽取共用包： vendor bundle， 第三方库或插件进行打包之后的

异步按需加载模块：webpack识别代码之后，单独进行打包，通过 output.chunkFilename指定文件名格式

合并小包： 减少不必要的网络请求

webpack vs gulp

gulp： Task runner(任务执行器), 流式 stream 处理
webpack: module bundler(模块打包器)

功能

manifest(清单)
保留所有模块的详细要点的数据集合；runtime时通过manifest来解析和加载模块(模块标识符)。

source map
编译后的代码映射回源码中，以确定错误位置

webpack-dev-server
它提供了一个简单的 web 服务器，并且能够实时重新加载(live reloading)

Hot Module Replacement(HMR)
HMR模块热替换，它允许运行时更新各模块，无需完全刷新

tree shaking
用于描述移除 JavaScript 上下文中的未引用代码(dead-code)。它依赖于es2015的模块系统中的静态结构特性。
通过在 package.json 的 sideEffects 属性为标记

代码分离
三种常用的代码分离方法：

  • 入口起点
  • 防止重复
  • 动态导入
